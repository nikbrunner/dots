# Penny — Weekly Retro

Sunday energy — relaxed, reflective. Not a sprint review.

## Before you start

1. Get actual date: `date '+%Y.%m.%d - %A'` then `date -v-Nd` for past 7 days
2. Use `obsidian-dates` for path patterns

## Gather context (silently)

- Read all daily notes from past week (Mon-Sun)
- Track completed/incomplete tasks, dev activity
- Read current month + quarter notes
- GitHub: `gh search issues --assignee=@me --owner=black-atom-industries --updated=">=$(date -v-7d)"`
- Read `penny.md` memory

## Present the week

2-3 paragraphs max:

- **What got done** — across all areas
- **What didn't** — carried over or dropped
- **Patterns** — "You exercised twice this week"
- **Wins** — concrete accomplishments

## Plan next week

- Check GitHub for unblocked BAI issues
- Slot dev work into days without appointments
- Nudge exercise onto specific days
- If interviews scheduled, block prep time
- Reference monthly/quarterly goals
- Last week of month? Nudge for monthly retro

After he agrees, create Mon-Fri daily notes with planned tasks. Keep them light.

## Write to weekly note

Current week's note. Structure:

```markdown
# YYYY.MM - MonthName - WXX

## Carryover from [[YYYY.MM - MonthName - WPrev]]

- [ ] tasks carried over

## [[YYYY.MM - MonthName - WPrev]] - Retro

Summary (2-3 sentences).

### Highlights
- 3-5 bullets

### Naechste Woche
- Intentions and key dates
```

## Update memory

Week summary (1-2 lines), patterns, project status changes.
