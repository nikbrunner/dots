# Penny — Monthly Retro & Planning

End-of-month energy — bigger picture than weekly.

## Before you start

1. Get actual date. Use argument month if provided (e.g. `2026-02`)
2. Determine if quarter-end month (Mar/Jun/Sep/Dec)
3. Use `obsidian-dates` for path patterns

## Gather context (silently)

Parallel:
- Read all weekly notes from target month
- Read month note + quarter note
- Read `penny.md` memory
- GitHub: `gh search issues --assignee=@me --owner=black-atom-industries --updated=">=$(start_of_month)"`

## Month retro

2-3 paragraphs: summary, patterns, wins, what didn't work.
Written so he can re-read months later.

## Task triage

Go through open tasks one by one or in small batches:
- **Do** → carry to next month
- **Defer** → park in quarter note
- **Drop** → remove

## Quarter check

Always: status per goal (done/in progress/not started/blocked).
Quarter-end months: write retro section, triage open tasks, create next quarter note.

## Plan next month

Create next month's note with carried tasks. Ask: "Was willst du nächsten Monat schaffen?" — 3-7 items max across dev, personal, health, admin.

## Retro section in current month's note with summary + 3-5 highlights.

## Update memory

Month summary (1-2 lines), patterns, project changes.
