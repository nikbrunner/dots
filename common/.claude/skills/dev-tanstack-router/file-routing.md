# File-Based Routing

## File Naming Conventions

| Pattern | Example | Purpose |
|-|-|-|
| `__root.tsx` | `src/routes/__root.tsx` | Root layout, always rendered, no path |
| `index.tsx` | `src/routes/index.tsx` | Index route (matches parent path exactly) |
| `about.tsx` | `src/routes/about.tsx` | Static route `/about` |
| `$param.tsx` | `src/routes/posts/$postId.tsx` | Dynamic segment `/posts/:postId` |
| `_layout.tsx` | `src/routes/_authenticated.tsx` | Pathless layout (wraps children, no URL path) |
| `(group)/` | `src/routes/(admin)/` | Route group (organization only, excluded from URL) |
| `.lazy.tsx` | `src/routes/about.lazy.tsx` | Manual code split (non-critical parts) |
| `-prefix` | `src/routes/-components/` | Excluded from routing entirely |

## Directory Structure Example

```
src/routes/
├── __root.tsx                    # App shell, persistent layout
├── index.tsx                     # /
├── about.tsx                     # /about
├── _authenticated.tsx            # Pathless layout (auth guard)
├── _authenticated/
│   ├── dashboard.tsx             # /dashboard (protected)
│   └── settings.tsx              # /settings (protected)
├── posts/
│   ├── index.tsx                 # /posts
│   └── $postId.tsx               # /posts/:postId
└── -components/                  # Shared route components (not a route)
    └── sidebar.tsx
```

## Route Tree Generation

`routeTree.gen.ts` is auto-generated by the bundler plugin (Vite/Webpack/Rspack). Never edit manually -- it regenerates on file changes and provides full type inference.

## Code Splitting

**Automatic (recommended):**

```ts
// vite.config.ts
import { tanstackRouter } from '@tanstack/router-plugin/vite'

export default defineConfig({
  plugins: [
    tanstackRouter({ autoCodeSplitting: true }),
    react(),
  ],
})
```

**Manual** (only when automatic splitting is unavailable):

Split route into `route.tsx` (critical: params, loaders) + `route.lazy.tsx` (non-critical: components).

## Type Safety Setup

Register the router with TypeScript for global type inference:

```tsx
// src/router.ts
import { createRouter } from '@tanstack/react-router'
import { routeTree } from './routeTree.gen'

const router = createRouter({ routeTree })

declare module '@tanstack/react-router' {
  interface Register {
    router: typeof router
  }
}
```

This enables type-safe `<Link>`, `useNavigate()`, and all route-scoped hooks throughout the app.
