Always use wikilinks when referencing Obsidian notes (e.g. `[[Erfolgsjournal]]` not just "Erfolgsjournal"). Nik called this out explicitly — "dafür sind die wikilinks da."

**Why:** Obsidian's linking is the backbone of the vault. Unlinked references are dead ends.

**How to apply:** Any time you mention an Obsidian note by name, wrap it in `[[]]`.

---

Don't use `>` blockquotes for email/message drafts that Nik needs to copy. Use plain text with `---` separators instead.

**Why:** Blockquotes are hard to copy cleanly from the terminal. Nik asked explicitly: "Kannst du bitte solche Text immer ohne die > quotes mir geben?"

**How to apply:** Email/message drafts → plain text between `---` lines. Blockquotes only for quoting others in notes.

---

When something is completed (e.g. "hab's geschickt"), immediately check it off in the Daily Note. Don't wait to be asked.

**Why:** Nik said "Immer wenn sowas ist kannst du eigentlich den Punkt in der Daily Note schon abkreuzen."

**How to apply:** Any time Nik confirms completing a task that's in the DN, edit the `- [ ]` to `- [x]` immediately.

---

Tasks for tomorrow go in tomorrow's Daily Note, not today's.

**Why:** Nik corrected "das macht mehr Sinn das eher in die morgige DN zu schreiben" and "Du wirst dich so nicht erinnern" — create the note now, don't defer.

**How to apply:** Future tasks → create the target day's note immediately and add the task there.
